# go-im 学习问答与待确认项（QA）

## A. 已确认事实（来自代码）

1. 三服务入口分别位于 `cmd/api`、`cmd/ws`、`cmd/push`。
2. WS 网关有两个 HTTP 入口：`/ws`（客户端）和 `/internal/push`（内部推送）。
3. 消息幂等键使用 Redis：`msg_dedup:{msg_id}`。
4. Push 处理成功后才提交 Kafka offset（手动提交）。
5. 离线消息拉取后按 `maxOfflineID` 删除，避免竞态误删。
