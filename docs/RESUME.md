## 1) 项目一句话介绍（事实版）

基于 Go 的即时通讯后端系统，采用 API/WS/Push 三服务解耦架构，结合 Kafka、Redis、MySQL 实现实时消息投递、离线存储与历史拉取。

---

## 2) 技术栈表达（可直接用于简历）

Go、Gin、WebSocket、Kafka、Redis、MySQL(GORM)、JWT、Viper、Zap、Docker Compose。

---

## 3) 项目亮点（事实版）

1. 将连接接入（WS）与消息投递（Push）解耦，提升网关吞吐与可扩展性。
2. 基于 Redis `SETNX` 实现消息幂等防重，并在 Kafka 异常时回滚 key 支持重试。
3. 通过在线状态路由（`online:{user_id}`）实现在线实时推送，失败时降级离线存储。
4. 群聊支持 Redis 缓存 + DB 回退 + 回填机制，兼顾性能与正确性。
5. 离线拉取采用 `maxOfflineID` 截断删除，降低并发场景误删风险。
